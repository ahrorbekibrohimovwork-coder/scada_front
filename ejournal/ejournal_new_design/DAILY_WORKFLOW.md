# Ежедневный цикл работ в системе электронного наряда-допуска

## Обзор

Добавлена полная система управления ежедневным циклом выполнения работ по наряду-допуску с поддержкой многодневных работ, продления наряда и контроля завершения.

---

## Новые компоненты

### 1. **DailyWorkFlow** (`src/app/components/permit/DailyWorkFlow.tsx`)

Центральный компонент управления ежедневным циклом работ.

**Возможности:**
- Отображение текущего статуса дня (в работе / завершён)
- Счётчик дней работы
- Контроль сроков с предупреждением о просрочке
- Информация о последнем инструктаже
- Кнопки действий в зависимости от статуса

**Доступные действия:**
- **Завершить работы дня** — подпись окончания рабочего дня (Производитель/Наблюдающий)
- **Проверка завершения работ** — запуск workflow для определения дальнейших действий
- **Начать следующий день** — создание нового ежедневного инструктажа
- **Продлить наряд** — открытие формы продления с валидацией

**Отображается для:**
- Производителя работ (foreman)
- Наблюдающего (observer)

**Статусы наряда:**
- `in_progress` — работы ведутся
- `admitted` — допуск к работе получен
- `daily_ended` — работы дня завершены

---

### 2. **ExtendPermitModal** (`src/app/components/permit/ExtendPermitModal.tsx`)

Модальное окно для продления наряда с умной валидацией.

**Функции:**
- Отображение текущей информации о наряде
- Расчёт оставшихся дней до лимита (15 дней)
- Валидация новой даты:
  - Не раньше текущей даты окончания
  - Не позже максимально допустимой (15 дней от начала работ)
- Предварительный расчёт продления
- Информация о требовании подписи ЭЦП

**Параметры:**
- `workStartDateTime` — дата начала работ
- `currentEndDateTime` — текущая дата окончания
- `extensionCount` — количество предыдущих продлений
- `onExtend` — callback при подтверждении продления

## Интеграция в WorkPermitDetail

### Вкладка "Инструктажи"

На вкладке "Инструктажи" (`tab === 'briefings'`) теперь отображается:

1. **Панель DailyWorkFlow** (только для Производителя/Наблюдающего):
   - Показывается при статусах: `in_progress`, `admitted`, `daily_ended`
   - Содержит все управляющие кнопки для ежедневного цикла

2. **Таблица инструктажей** (`DailyBriefingTable`):
   - История всех ежедневных инструктажей
   - Подписи допускающего, ответственного, членов бригады
   - Время начала и окончания каждого дня

## Workflow ежедневного цикла

### Сценарий 1: Стандартный рабочий день

```
1. Допускающий создаёт запись инструктажа
   ↓
2. Допускающий подписывает инструктаж (ЭЦП)
   ↓
3. Производитель/Наблюдающий подписывает инструктаж
   ↓
4. Члены бригады подписывают (только при первом допуске)
   ↓
5. Работы ведутся (статус: in_progress)
   ↓
6. Производитель/Наблюдающий завершает работы дня
   ↓
7. Статус → daily_ended
```

### Сценарий 2: Начало нового дня

```
Условие: статус = daily_ended И последний инструктаж подписан

1. Производитель нажимает "Проверка завершения работ"
   ↓
2. Система показывает WorkCompletionFlow:
   - "Работы завершены?" → Нет
   - "В срок?" → Да
   - "Есть изменения?" → Нет
   - "Изменения в бригаде?" → Нет
   ↓
3. Производитель нажимает "Начать следующий день"
   ↓
4. Система создаёт новую запись инструктажа
   ↓
5. Цикл повторяется с п.2 Сценария 1
```

### Сценарий 3: Продление наряда

```
Условие: работы не завершены, но срок истекает

1. Производитель нажимает "Продлить наряд"
   ↓
2. Открывается ExtendPermitModal
   ↓
3. Система проверяет:
   - Текущую длительность < 15 дней
   - Количество оставшихся дней
   ↓
4. Производитель вводит новую дату
   ↓
5. Валидация:
   - Дата > текущее окончание
   - Общая длительность ≤ 15 дней
   ↓
6. Подписание ЭЦП (Выдающий наряд)
   ↓
7. Наряд продлён, работы продолжаются
```

### Сценарий 4: Завершение работ и закрытие

```
1. Производитель нажимает "Проверка завершения работ"
   ↓
2. Система показывает WorkCompletionFlow:
   - "Работы завершены?" → Да
   ↓
3. Переход к форме закрытия наряда
   ↓
4. Производитель заполняет форму и подписывает ЭЦП
   ↓
5. Если назначен Ответственный руководитель:
   - Он подписывает закрытие (шаг 2/2)
   ↓
6. Статус → closed
```

## Валидации и ограничения

### Максимальная длительность: 15 дней

- Отсчёт от `workStartDateTime`
- Включает все продления
- При достижении лимита — кнопка продления заблокирована

### Перед началом нового дня

- Текущий день должен быть завершён
- Последний инструктаж должен иметь подпись окончания (`endSignature`)

### Перед закрытием наряда

- Допускающий должен проверить рабочие места
- Инструктаж должен быть проведён
- Все обязательные подписи должны быть получены

### При изменении бригады >50%

- Наряд должен быть закрыт
- Проверка происходит автоматически в WorkCompletionFlow

- Подсказки о лимитах и требованиях
- Предупреждения о просрочке
- Предварительный расчёт при продлении

### Цветовое кодирование

- **Зелёный** — работы ведутся, всё в норме
- **Жёлтый/Янтарный** — день завершён, требуется действие
- **Оранжевый** — продление, внимание к срокам
- **Красный** — просрочка, критические действия

## Подписи ЭЦП в ежедневном цикле

### Ежедневный инструктаж

1. **Допускающий** (`admitterSignature`)
   - Проводит инструктаж
   - Первая подпись

2. **Производитель/Наблюдающий** (`responsibleSignature`)
   - Подтверждает получение инструктажа
   - Если назначен Наблюдающий — он подписывает
   - Иначе подписывает Производитель

3. **Члены бригады** (`brigadeSignatures`)
   - Подписывают только при ПЕРВОМ допуске
   - При последующих днях — не требуется

### Окончание дня

- **Производитель/Наблюдающий** (`endSignature`)
- Подтверждает вывод бригады с рабочего места

### Продление наряда

- **Выдающий наряд** (`issuerSignature`)
- Подтверждает изменение срока действия

## Технические детали

### Структура данных DailyBriefing

```typescript
interface DailyBriefing {
  id: string;
  isFirst: boolean;                    // Первичный допуск?
  workLocationName: string;             // Место работы
  briefingDateTime: string;             // Дата/время инструктажа
  admitterSignature?: EDSSignature;     // Подпись допускающего
  responsibleSignature?: EDSSignature;  // Подпись ответственного
  brigadeSignatures: {                  // Подписи бригады
    memberId: string;
    memberName: string;
    sig: EDSSignature;
  }[];
  endDateTime?: string;                 // Время окончания дня
  endSignature?: EDSSignature;          // Подпись окончания
}
```

### Методы WorkPermitContext

- `createBriefing(id, briefing)` — создание записи инструктажа
- `signBriefingAdmitter(id, briefingId, sig)` — подпись допускающего
- `signBriefingResponsible(id, briefingId, sig)` — подпись ответственного
- `signBriefingMember(id, briefingId, memberId, name, sig)` — подпись члена бригады
- `endDailyWork(id, briefingId, endDateTime, sig)` — завершение дня
- `extendPermit(id, newEndDateTime, sig)` — продление наряда

## Пример использования

### Запуск нового дня (Производитель работ)

```tsx
<DailyWorkFlow
  permit={permit}
  isForeman={true}
  isObserver={false}
  onStartNewDay={() => {
    const brief: DailyBriefing = {
      id: `db_${Date.now()}`,
      isFirst: false,
      workLocationName: lastBriefing?.workLocationName || '',
      briefingDateTime: new Date().toISOString(),
      brigadeSignatures: [],
    };
    ctx.createBriefing(permit.id, brief);
  }}
  onExtendPermit={() => setModal('extend_form')}
  onEndDay={() => {
    ctx.endDailyWork(permit.id, lastBriefing.id, 
      new Date().toISOString(), makeSig());
  }}
  onCheckCompletion={() => setShowCompletionFlow(true)}
/>
```

### Продление наряда

```tsx
<ExtendPermitModal
  workStartDateTime="2026-04-10T08:00:00"
  currentEndDateTime="2026-04-15T18:00:00"
  extensionCount={0}
  onClose={() => setModal(null)}
  onExtend={(newEndDateTime) => {
    // Сохранить дату и открыть ЭЦП
    setExtendDate(newEndDateTime);
    setModal('extend_sign');
  }}
/>
```
