## ✅ Исправление логики автоматического изменения статусов

### Проблема

Ранее статус наряда не менялся автоматически после получения всех необходимых подписей. Требовалось ручное изменение.

### Решение

**Автоматический переход: `admitted` → `in_progress`**

Статус автоматически меняется на `in_progress` когда:

1. **Первичный допуск (isFirst: true):**
   - ✅ Допускающий подписал инструктаж
   - ✅ Производитель/Наблюдающий подписал инструктаж
   - ✅ **ВСЕ** члены бригады подписали инструктаж

2. **Последующие дни (isFirst: false):**
   - ✅ Допускающий подписал инструктаж
   - ✅ Производитель/Наблюдающий подписал инструктаж
   - Подписи членов бригады не требуются

### Изменённые методы

**`signBriefingResponsible`** — автоматически проверяет все подписи:
```typescript
const allMembersSigned = !briefing?.isFirst ||
  p.brigadeMembers.filter(m => m.isActive).every(m =>
    briefing.brigadeSignatures.some(s => s.memberId === m.id)
  );

const newStatus = allMembersSigned ? 'in_progress' : 'admitted';
```

**`signBriefingMember`** — триггерит переход после последней подписи члена бригады:
```typescript
const allMembersSigned = p.brigadeMembers.filter(m => m.isActive).every(m =>
  updatedSignatures.some(s => s.memberId === m.id)
);

const newStatus = (briefing?.isFirst && ... && allMembersSigned)
  ? 'in_progress'
  : p.status;
```

## 🔄 Переработка логики продления наряда

### Старая логика (УДАЛЕНА)

❌ Продление зависело от длительности наряда от начала работ  
❌ Лимит: 15 дней от `workStartDateTime`  
❌ Можно было продлевать несколько раз  

### Новая логика

✅ **Продление НЕ зависит от оставшегося срока**  
✅ **Доступно в любой момент пока наряд активен**  
✅ **Можно продлить только ОДИН раз**  
✅ **Максимум: 15 календарных дней от ТЕКУЩЕЙ даты**  

### Правила продления

| Параметр | Значение |
|----------|----------|
| Количество продлений | **1** (только один раз) |
| Максимальный срок | **15 дней от текущей даты** |
| Точка отсчёта | Дата нажатия кнопки "Продлить" |
| Минимальная дата | Сегодня + 1 день |
| Максимальная дата | Сегодня + 15 дней |

## 👥 Логика работы с членами бригады

### Подписи членов бригады

**Правило:** Члены бригады подписывают инструктаж **ТОЛЬКО ОДИН РАЗ** при первичном допуске.

#### Первичный допуск (День 1, isFirst: true)

1. Допускающий подписывает инструктаж ✅
2. Производитель/Наблюдающий подписывает ✅
3. **Каждый** член бригады подписывает ✅
4. После всех подписей → автоматически `in_progress`

#### Последующие дни (День 2+, isFirst: false)

1. Допускающий подписывает инструктаж ✅
2. Производитель/Наблюдающий подписывает ✅
3. Члены бригады **НЕ** подписывают ❌
4. Сразу после подписи ответственного → `in_progress`

### UI отображение в DailyWorkJournal

**День 1 (первичный допуск):**
```
Члены бригады:
  Новиков В.А.        [ЭЦП] или ✔️
  Каримов А.Б.        [ЭЦП] или ✔️
  Садыков Р.И.        [ЭЦП] или ✔️
```

**День 2+ (последующие дни):**
```
Члены бригады:
  Новиков В.А.        —
  Каримов А.Б.        —
  Садыков Р.И.        —
```

Прочерк `—` означает "не требуется" (подписано при первичном допуске).

## 📅 Состав бригады по дням

### Новая логика (TODO - в разработке)

Каждый день имеет свой **фиксированный** состав бригады:

```typescript
interface DailyBriefing {
  id: string;
  isFirst: boolean;
  workLocationName: string;
  briefingDateTime: string;
  
  // Состав бригады на этот день
  dayBrigadeMembers?: BrigadeMember[];  // ← НОВОЕ
  
  admitterSignature?: EDSSignature;
  responsibleSignature?: EDSSignature;
  brigadeSignatures: {...}[];
  endDateTime?: string;
  endSignature?: EDSSignature;
}
```

### Правила

1. **Новый день** — копируется состав из предыдущего дня
2. **Добавление члена** — появляется только в текущем и последующих днях
3. **Удаление члена** — удаляется из текущего и последующих дней
4. **Предыдущие дни** — состав НЕ меняется (исторические данные)

### UI (в разработке)

```
День 1: 17.04.26
  ✅ Новиков В.А. (III)
  ✅ Каримов А.Б. (III)
  ✅ Садыков Р.И. (II)

День 2: 18.04.26  ← добавлен Иванов С.П.
  ✅ Новиков В.А. (III)
  ✅ Каримов А.Б. (III)
  ✅ Садыков Р.И. (II)
  ✅ Иванов С.П. (III)  ← новый член
```

## 🚀 Как тестировать

### 1. Автоматический переход статусов

```
1. Войти как Допускающий (sidorov / 1234)
2. Открыть наряд в статусе "Рабочие места одобрены"
3. Вкладка "Инструктажи" → Подписать инструктаж
4. Войти как Производитель (kozlov / 1234)
5. Подписать инструктаж → проверить что член бригады может подписать
6. Войти как Член бригады (novikov / 1234)
7. Подписать инструктаж
8. Повторить для всех членов бригады
9. ✅ После последней подписи статус → "В работе"
```

### 2. Логика продления (один раз)

```
1. Войти как Производитель (kozlov / 1234)
2. Открыть наряд в статусе "В работе"
3. Вкладка "Инструктажи" → Нажать "Продлить наряд"
4. Указать дату (макс. +15 дней от сегодня)
5. Подписать ЭЦП
6. ✅ Наряд продлён
7. Попытаться продлить ещё раз
8. ✅ Кнопка заблокирована: "Продление использовано"
```

### 3. Подписи членов бригады (только первый день)

```
1. Создать новый наряд с 3 членами бригады
2. Пройти workflow до первичного допуска
3. ✅ Члены бригады подписывают в День 1
4. Завершить День 1
5. Создать День 2 (новый инструктаж)
6. ✅ Члены бригады НЕ подписывают в День 2
7. Статус → сразу "В работе" после подписи ответственного
```
