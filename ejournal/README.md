# Электронный журнал нарядов-допусков (Electronic Journal)

Информационная система для автоматизации процесса выписки, согласования, допуска и закрытия нарядов-допусков на производство работ в электроустановках. Система обеспечивает полную прослеживаемость процесса и легитимность действий через электронные цифровые подписи (ЭЦП).

- **Framer Motion** для анимаций цепочки подписей

- **Python 3.9+**
- **FastAPI** (высокопроизводительный веб-фреймворк)

## 📊 Структура базы данных (PostgreSQL)

### 1. Таблица `work_permit` (Основная информация о наряде)
| Колонка | Тип | Описание |
| :--- | :--- | :--- |
| `id` | Integer (PK) | Уникальный номер наряда |
| `organization` | String | Организация |
| `department` | String | Подразделение |
| `start_time` | DateTime | Время начала работ |
| `end_time` | DateTime | Время окончания работ |
| `work_description` | Text | Содержание работ |
| `special_instructions`| Text | Особые указания |
| `status` | String | Текущий статус (DRAFT, IN_PROGRESS и т.д.) |
| `issuer_signature` | JSONB | Подпись выдающего наряд |
| `dispatcher_signature`| JSONB | Разрешение на подготовку (Диспетчер) |
| `admitter_workplace_signature`| JSONB | Подпись допускающего (рабочие места) |
| `workplace_verifier_signature` | JSONB | Подпись проверяющего (Наблюдающий/Производитель) |
| `live_parts` | String | Части, оставшиеся под напряжением |

### 2. Таблица `daily_briefings` (Инструктажи и ежедневные допуски)
Хранит историю ежедневных допусков и подписи участников инструктажа.
- `id`: Уникальный ID записи.
- `permit_id`: Ссылка на наряд.
- `is_first`: Флаг первичного инструктажа.
- `admitter_signature`: Подпись допускающего.
- `responsible_signature`: Подпись ответственного лица (Производитель/Наблюдающий).
- `brigade_signatures`: JSON-список подписей всех членов бригады.
- `end_datetime`: Время завершения работ дня.

### 3. Таблица `app_users` (Пользователи системы)
- `id`: UUID пользователя.
- `login`: Логин для входа.
- `password`: Хэш пароля.
- `full_name`: ФИО.
- `role`: Системная роль (admin, dispatcher, admitter и т.д.).
- `electrical_group`: Группа по электробезопасности.

### 4. Дополнительные таблицы
- `permit_events`: История всех действий с нарядом (лог событий).
- `return_comments`: Комментарии при возврате наряда на доработку.
- `brigade_members`: Список членов бригады, назначенных в наряд.
- `permit_extensions`: Записи о продлении срока действия наряда.

## 🔄 Жизненный цикл наряда (Workflow)

Процесс проходит через следующие ключевые статусы:

1.  **DRAFT (Черновик)**: Создание наряда выдающим.
2.  **PENDING_DISPATCHER**: Ожидание разрешения от главного диспетчера.
3.  **PREPARING_WORKPLACES**: Подготовка рабочих мест (Помощник ГД).
4.  **PENDING_ADMITTER**: Допускающий проверяет подготовку на месте.
5.  **WORKPLACE_APPROVED**: Рабочие места проверены и приняты Производителем.
6.  **ADMITTED**: Проведен инструктаж, допускающий расписался.
7.  **IN_PROGRESS (В работе)**: Все участники подписали инструктаж. Работа разрешена.
8.  **DAILY_ENDED**: Работы дня завершены (временная остановка).
9.  **CLOSING**: Наряд подан на закрытие Производителем.
10. **CLOSED**: Наряд полностью закрыт Ответственным руководителем.

- Node.js 18+
- Python 3.9+

```bash
cd tb_backend
```

## 🔐 Безопасность и ЭЦП
Все подписи в системе хранятся в формате JSONB, включая:
- Идентификатор пользователя.
- ФИО и должность на момент подписания.
- Временную метку (Timestamp).
- Данные сертификата (имитация для учебного проекта).

Доступ к действиям строго ограничен ролями, назначенными в конкретном наряде.
